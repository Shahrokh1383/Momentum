
### 🛠️ Phase 1: Foundation, Auth & PWA Core
در این فاز پایه‌های سیستم کاربران، اشتراک و تنظیمات را می‌ریزیم. بنابراین فقط Command های مرتبط با این جداول باید ایجاد شوند.

**۱. `subscriptions:clean-expired`**
*   **دلیل پیاده‌سازی در این فاز:** جدول `subscriptions` در همین فاز ساخته می‌شود. اگر کاربری اشتراکش تمام شود اما وضعیتش در دیتابیس روی `active` بماند، در فازهای بعدی به اشتباه به امکانات Premium دسترسی خواهد داشت (Bypass امنیتی).
*   **نکته سینیور:** در این Command، پس از آپدیت استاتوس به `expired`، حتماً یک Event (مثلاً `SubscriptionExpired`) را فایر کن تا در فاز ۴ اگر سیستم نوتیفیکیشن آماده بود، به کاربر اخطار داده شود.

**۲. ایجاد فایل `config/momentum.php` (نسخه اولیه)**
*   **دلیل پیاده‌سازی در این فاز:** برای جدا کردن Business Logic از کد، از روز اول باید این فایل وجود داشته باشد.
*   **محتوای این فاز:** در این فاز فقط مقادیر مربوط به Quota های پلن‌ها (که در جدول `plans` هم هست اما ممکن است بخواهی Override کند) و تنظیمات پایه سیستم را در آن قرار بده. مثلاً:
    ```php
    return [
        'subscription' => [
            'grace_period_days' => 3, // روز تضمین بازگشت وجه
        ],
    ];
    ```

### 🛠️ Phase 2: Enhanced Data Model & Core Tracking
در این فاز هسته اصلی یعنی عادت‌ها، لاگ‌ها و استریک‌ها شکل می‌گیرند. این فاز بارِ سنگینِ Cron Job های سیستم را به دوش می‌کشد.

**۱. `habits:mark-missed`**
*   **دلیل پیاده‌سازی در این فاز:** جداول `habits` و `habit_logs` در همین فاز ایجاد می‌شوند و منطق `isDueToday` و `user_now()` در کنترلرها تکمیل می‌شود. بدون این Command، استریک کاربران هرگز قطع نمی‌شود و سیستم Gamification فاز ۴ کاملاً باگ‌دار خواهد بود.
*   **نکته سینیور:** حتماً از `chunkById` استفاده کن تا در صورت افزایش کاربران، Memory Leak نداشته باشی. مطمئن شو منطق Timezone با هلپر `user_now()` که در این فاز می‌نویسی ۱۰۰٪ سینک باشد.

**۲. `reminders:dispatch` (نسخه اولیه - بدون Real-time)**
*   **دلیل پیاده‌سازی در این فاز:** فیلد `reminder_time` در جدول `habits` در این فاز اضافه می‌شود. این Command باید در این فاز نوشته شود تا کار کند.
*   **نکته سینیور:** در این فاز Laravel Reverb را هنوز پیاده نکرده‌ایم. پس این Command در این فاز فقط باید نوتیفیکیشن‌ها را در دیتابیس (جدول `notifications` که در فاز ۴ می‌آید) یا در یک Log ذخیره کند. در فاز ۴ این Command را ریفکتور می‌کنی تا Event ها را به Reverb广播 (Broadcast) کند.

**۳. به‌روزرسانی `config/momentum.php`**
*   **محتوای این فاز:** اضافه کردن تنظیمات مربوط به استریک و عادت:
    ```php
    'habits' => [
        'missed_check_tolerance_minutes' => 15,
    ],
    'streaks' => [
        'freeze_enabled' => true,
    ]
    ```

### 🛠️ Phase 3: Advanced Habit Tracking & Journal
در این فاز، منطق آفلاین (Offline Sync) و لاگ‌های پیچیده‌تر اضافه می‌شود. هیچ Command جدیدی در لیست ما وجود ندارد که مربوط به این فاز باشد، اما کار نگهداری داریم:

*   **نکته سینیور:** باید Command `habits:mark-missed` فاز ۲ را آپدیت کنی تا تایپ‌های جدید `skipped` و منطق `skip_reason` را به درستی در محاسبه استریک در نظر بگیرد (طبق سناریوی StreakService در فاز ۳).

### 🛠️ Phase 4: Gamification, Social & Real-time
این فاز زمان شکوفایی سیستم رویدادها و صف‌هاست. جدول `notifications` و `leaderboard_entries` اینجا ساخته می‌شوند.

**۱. `leaderboard:compute`**
*   **دلیل پیاده‌سازی در این فاز:** جداول `xp_transactions`، `levels` و `leaderboard_entries` در همین فاز ایجاد می‌شوند. محاسبه قبلی (Tie-breaker) نیازمند دیتای XP است که از فاز ۴ به بعد تولید می‌شود.
*   **نکته سینیور:** حتماً از Transaction های دیتابیس استفاده کن و پاک کردن رکوردهای قبلی دوره (مثلاً هفته گذشته) و درج رکوردهای جدید را در یک DB Transaction انجام بده تا در صورت Fail کردن Job، دیتای leaderboard از بین نرود.

**۲. ریفکتور `reminders:dispatch`**
*   **عملیات:** حالا که Reverb نصب است، به جای ذخیره ساده، در این Command وقتی زمان فرا رسید، یک `BroadcastEvent` را فایر کن تا از طریق WebSocket به فرانتند پوش شود.

**۳. به‌روزرسانی `config/momentum.php`**
*   **محتوای این فاز:** اضافه کردن تنظیمات گیمیفیکیشن و گروه:
    ```php
    'groups' => [
        'completion_threshold_percentage' => 80, // 80% اعضا باید لاگ بزنند
    ],
    'xp' => [
        'early_bird_hour' => 8,
        'default_habit_weight' => 10,
    ]
    ```

### 🛠️ Phase 5: Intelligence, Insights & Reports
آخرین حلقه زنجیر، تولید گزارش و هوش مصنوعی ساده است که به شدت به Job ها و صف‌ها وابسته است.

**۱. `insights:generate`**
*   **دلیل پیاده‌سازی در این فاز:** جداول `insights` و `correlations` همین الان ایجاد می‌شوند. دیتای لاگ‌ها هم که از فاز ۲ آماده است.
*   **نکته سینیور:** طبق برنامه‌ریزی، این Command نباید پردازش را خودش انجام دهد. این Command فقط باید کاربران را `chunkById` کند و برای هر کاربر یک `GenerateUserInsightsJob` به صف (Database Queue) بفرستد. این معماری برای جلوگیری از OOM (Out of Memory) در آینده حیاتی است.

**۲. `reports:clean-old`**
*   **دلیل پیاده‌سازی در این فاز:** فایل‌های PDF و جدول `reports` در همین فاز متولد می‌شوند. بدون این Command، دیسک سرور شما پر از فایل‌های PDF متروکه خواهد شد.

**۳. `pdf-quota:reset-monthly`**
*   **دلیل پیاده‌سازی در این فاز:** ستون `pdfs_generated_this_month` در جدول `users` در همین فاز اضافه می‌شود. این Command در روز اول هر ماه باید اجرا شود تا سهمیه کاربران رایگان بازنشانی شود.

**۴. به‌روزرسانی نهایی `config/momentum.php`**
*   **محتوای این فاز:** تنظیمات الگوریتم‌های هوشمند:
    ```php
    'insights' => [
        'risk_weights' => [
            'late_nights' => 10,
            'weekend_misses' => 15,
            'recent_misses' => 20,
        ],
        'correlation_threshold' => 0.80,
        'correlation_min_sample' => 10,
    ],
    'reports' => [
        'retention_days' => 90,
        'free_tier_monthly_limit' => 1,
    ]
    ```

*   **Phase 5:** `insights:generate` + `reports:clean-old` + `pdf-quota:reset-monthly`
